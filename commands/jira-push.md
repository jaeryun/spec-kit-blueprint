# Blueprint Jira Push

Push the Epic → Story hierarchy from `blueprint.md` to Jira (create Epics, Stories, link them).

## Trigger

Automatically invoked by `after_blueprint_roadmap` hook.

## Purpose

After `/speckit.blueprint.roadmap` generates or updates the Epic/Story hierarchy, this hook ensures every Epic and Story has a corresponding Jira issue. It creates missing issues, updates existing ones, and stores the Jira issue keys back in the Blueprint files.

## Instructions

### Step 1: Read Configuration

Read `.specify/memory/blueprint.yml` for:

- `project_key` — Jira project key
- `epic_issue_type` — Epic issue type name (default: "Epic")
- `story_issue_type` — Story issue type name (default: "Story")

If `blueprint.yml` is missing or `project_key` is not set, log:

> `[jira-push] Skipped: blueprint.yml not configured. Run /speckit.blueprint.setup.`

Stop. Do not error.

---

### Step 2: Read Roadmap

Read the Blueprint files generated by roadmap:

- `docs/blueprint/blueprint.md` — extract Epic list (EP IDs, titles, scope) and Story list per Epic (ST IDs, titles, Key AC)

If the file is missing, log:

> `[jira-push] Skipped: roadmap files not found.`

Stop.

---

### Step 3: Create/Update Jira Epics

For each EP in `blueprint.md`:

1. Search Jira by label `blueprint-ep-[ID]` using `mcp-jira/search`.
2. If found: update the Epic with `mcp-jira/update_issue` if title or description changed.
3. If not found: create with `mcp-jira/create_issue` using this format:

```yaml
Project: [project_key]
Issue Type: [epic_issue_type]
Summary: [EP Title]
Description: [EP Scope]
Labels: blueprint, blueprint-ep-[ID]
```

4. Store the returned issue key (e.g., `PROJ-42`).

On any error (search, create, or update), log:

> `[jira-push] Epic [EP-ID] failed: [error message]. Continuing...`

Do not block subsequent Epics or Stories.

---

### Step 4: Create/Update Jira Stories

For each ST in `blueprint.md`:

1. Search Jira by label `blueprint-st-[ID]` using `mcp-jira/search`.
2. If found: update the Story with `mcp-jira/update_issue` if title, description, or Epic Link changed.
3. If not found: create with `mcp-jira/create_issue` using this format:

```yaml
Project: [project_key]
Issue Type: [story_issue_type]
Summary: [ST Title]
Description: [ST Overview]
Labels: blueprint, blueprint-st-[ID]
Epic Link: [parent Epic issue key]
```

4. Store the returned issue key.

On any error, log:

> `[jira-push] Story [ST-ID] failed: [error message]. Continuing...`

Do not block subsequent Stories.

---

### Step 5: Update Blueprint Files

For each successfully created or updated Jira issue, write the issue key back into the corresponding Blueprint file:

- `blueprint.md` — update the Jira link for each EP and ST
- `epics/[epic-slug]/[story-slug]/story.md` — update the Jira link in the header (e.g., `Jira: PROJ-43`). To find the correct directory, search under each `epics/*/` directory and match the Story ID in the `story.md` title.

If a file update fails, log:

> `[jira-push] Failed to write Jira key [KEY] to [file]: [error]. Continuing...`

---

### Step 6: Completion Log

Output a summary:

```text
[jira-push] Complete.

| Level | Created | Updated | Failed |
| --- | --- | --- | --- |
| Epics | [N] | [N] | [N] |
| Stories | [N] | [N] | [N] |
```

If any failures occurred, append:

```text
⚠️ Some issues could not be synced. Check logs above and re-run /speckit.blueprint.roadmap to retry.
```
